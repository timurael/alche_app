I · Preface

We started with a home screen.
We ended with a voice.

The home-screen study surfaced three patterns that could not be solved by a better home — they were written into every other screen too. So we wrote them out, in order.

The thesis

An editorial design language — Newsreader italic for display, Space Mono for overlines, 2-pixel sharp corners, hard drop shadows — cannot be applied to a screen after the fact. If the rhythm of the screen is still dashboard, no amount of font discipline will rescue it. The font becomes costume on a fundamentally corporate garment.

The three patterns the audit surfaced — token drift, voice absent between blocks, and fictional mono metadata as costume — are not independent. They share one root cause: every screen was built assuming the user wants to read a dashboard. Longevity isn't a dashboard. It's a practice. And a practice is conducted with a person, not a UI.

Strategy 1 — per-screen direction. Each screen picks the register that fits its primary intent. Three peers: Dialog-first (the screen speaks first, asks second, lists last), Signal-first (a single pattern carries the screen — a curve, a silhouette, a chart), Ritual-first (the next moment is the page — slot, step, threshold). Voice family stays shared via the narrator kit; the rule is per-screen fit, not one universal frame.

The rewrite is not a repaint. Many screens lose whole sections; some gain a voice where there was silence; all of them answer, in their first beat, the question the member was about to ask — and each in the register that question deserves.

Between the deck's first draft and today, one more voice joined the app: Lix, a droplet mascot hosting the onboarding in lowercase. The redesign learned to speak in two registers — Lix before the member signs, Alche after. What used to be eight onboarding screens is now one scripted conversation; the baton-pass to Alche is itself a 0.8-second black curtain.

Below: the shared narrator kit every screen uses, five groups of screens with a voice direction per group, and an implementation roadmap that sequences the remaining screens for the next sprints. Home, Lix, Profile, and Discover are already live.

II · Narrator kit

Seven patterns
that carry the voice.

Every mockup in this book is built from the same shared components. The consistency is not visual — it is grammatical.

Pattern 01
Context band

A single Space Mono row at the top of every screen. Date, moment, session count. Never "Good morning." Declarative, oriented.

.alche-context-band
Pattern 02
Read stamp

Two-by-two black-and-blue mark + "ALCHE · YOUR READ" label. Signals the narrator is about to speak.

.alche-read-stamp
Pattern 03
The read

Three italic Newsreader lines. Each sentence a breath. One phrase underlined in action-blue — the target of the primary action.

.alche-read
Pattern 04
Data strip

Space Mono line of real data — sleep, HRV, load, recovery. Never faux-technical. If it doesn't carry a value, it isn't here.

.alche-data-strip
Pattern 05
Interstitial

Between data blocks, the narrator speaks again. Short italic aside. The cure for silent stacking — used on almost every screen.

.alche-interstitial
Pattern 06
Previous echo

One tiny italic line carrying forward yesterday — or the previous step. Makes the screen feel like part of an ongoing conversation.

.alche-prev-echo
Pattern 07
Browse escape

A tiny mono link at the bottom: "Browse instead →". An honest exit for the member who didn't come to be read today.

.alche-browse-escape
Pattern 08
Lix voice

Pre-auth screens carry a separate register: lowercase, 4-14 words, warm-terse. Post-auth is editorial italic Alche. Two voices, one app.

.lix-line

VII · Group 03

Wellness features —
the body, narrated.

Numbers without a narrator are a missed moment. Every data block on these screens is prefaced by an observation. Evidence, not dashboard.

Voice direction

Observational, never clinical.

Never "indicates" / "demonstrates" / "diagnoses." Always "looks," "reads," "we see," "your body is." Glow Scan is a Glow Score, not a Health Score. Mock data is honest and consistently tagged. Sample Data is present but not frantic.

IX · Group 05

Services & rituals —
the transactional moments.

Where most apps lose their voice. Alche frames the transaction — one observational read, then the utility. The narrator is brief but present, even at check-in.

Voice direction

Framed, not narrated.

Transactional screens get the narrator's opening line and not much else. One primary action per section. No capsule pills, no fake metadata, no pastel rings. Health-safe language per app conventions.
