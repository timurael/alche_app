# Protocols — Swift Implementation Plan

## What changes (high level)

`ProtocolListView` is restructured so the **active protocol owns the screen**. Everything else demotes to "considered for you" (max 3) and a mono "Browse all protocols" escape hatch.

Top to bottom:

1. Top bar: back link + `DataSourceIndicator("Sample Data")`.
2. `AlcheReadStamp` + `AlcheReadCard(lines:)` — 2-line italic opening read.
3. **Active protocol hero block** — kicker ("ACTIVE · DAY 21 OF 42"), italic Newsreader name, hairline progress bar + mono count. NOT a card — it IS the screen.
4. Primary action `Open Today's Steps · N left` + secondary `Pause protocol`.
5. `AlcheInterstitial` narrator line transitioning to recommendations.
6. "CONSIDERED FOR YOU · 3" block — three italic line-item rows. Each: italic name, one-line rationale, mono duration + chevron.
7. `Browse all protocols →` escape hatch (opens full `ProtocolLibraryView` behind it).
8. Previous-echo line documenting adaptation ("You skipped the breathwork step twice. We adjusted around it.").

If no active protocol: the screen opens with a different read ("You're between protocols. Here are three I'd consider.") and the hero collapses into the considered block.

## View structure (target)

```swift
struct ProtocolListView: View {
    @State private var viewModel = ProtocolsViewModel()

    var body: some View {
        ScrollView {
            VStack(alignment: .leading, spacing: 0) {
                AlcheReadStamp(label: "ALCHE · YOUR PROTOCOL TODAY")
                AlcheReadCard(lines: viewModel.openingRead)

                if let active = viewModel.activeProtocol {
                    ActiveProtocolHero(protocol: active,
                                       stepsCompleted: viewModel.stepsCompletedToday(for: active),
                                       totalSteps: viewModel.totalStepsToday(for: active),
                                       day: viewModel.dayInProtocol(for: active),
                                       total: active.totalDays)
                    AlchePrimaryAction(label: "Open Today's Steps · \(viewModel.stepsLeft) left") {
                        viewModel.openTodaySteps()
                    }
                    AlcheSecondaryAction(label: "Pause protocol") { viewModel.pauseActive() }
                }

                AlcheInterstitial(voice: viewModel.considerationTransition)
                ConsideredList(items: viewModel.considered)

                Link("Browse all protocols →", destination: .protocolLibrary)
                    .alcheBrowseEscapeStyle()

                PreviousEchoRow(tag: viewModel.prevEchoTag,
                                echo: viewModel.prevEchoText)
            }
        }
        .background(Color.alcheBackground)
        .toolbar {
            ToolbarItem(placement: .topBarTrailing) {
                DataSourceIndicator("Sample Data")
                    .opacity(viewModel.isUsingMockData ? 1 : 0)
            }
        }
        .navigationBarTitleDisplayMode(.inline)
    }
}
```

### Sub-views

- `ActiveProtocolHero` — mono kicker ("ACTIVE · DAY \(day) OF \(total)"), italic `.alcheDisplayL` name, `AlcheHairlineProgress(value:)` + mono count ("2 OF 4 TODAY"). No card chrome.
- `ConsideredList` — header with "CONSIDERED FOR YOU · 3" overline, then 3 `ConsideredRow`s. Each row: italic name (`.alcheDisplayS`), one-line body (`.alcheCaption`), mono duration + chev (right column).
- `AlcheHairlineProgress(value:)` — 3px high, `Color.alcheMuted` track, `Color.alcheInk` fill.
- `ConsideredRow` — sharp-edge tap target, full-width button that pushes `ProtocolDetailView`.

## ViewModel changes

Edits to `ProtocolsViewModel`:

- Add `openingRead: [String]` — generated by `ProtocolNarrator`. Two lines max. Situates the active protocol with a real observation ("Sleep is 14% deeper").
- Add `considered: [Protocol]` — max 3, filtered by member's wellness goals, excluding the active, excluding locked-by-tier. Ranked by goal-match score.
- Add `considerationTransition: String` — interstitial line. If active: "When you finish this one, here's what I'd consider next". If no active: "You're between protocols. These three fit what you chose."
- Add `stepsLeft: Int` — total steps today minus completed.
- Add `dayInProtocol(for:) -> Int` and `totalDays` on `Protocol`.
- Add `prevEchoTag: String`, `prevEchoText: String` — derived from `viewModel.adaptationHistory`. If no history, omit the echo row entirely (don't render empty chrome).
- **Remove** `selectedGoal` and the whole goal filter array — goal filtering is implicit in `considered` ranking.
- **Remove** the locked-tier rendering path in the main view body. Locked protocols remain in the full `ProtocolLibraryView` only.

## Data dependencies

- `Protocol` model needs: `totalDays: Int`. Compute from protocol start date + duration if field absent.
- `ProtocolsViewModel.adaptationHistory: [AdaptationEvent]` — events that capture skipped/swapped steps. New lightweight model; can begin as a computed property over `MockProtocolService` history.
- `ProgressService` tap for "Sleep is 14% deeper" style observations (already available via `ProgressViewModel`).

## Accessibility notes

- Active hero: combined `.accessibilityLabel("Active protocol: Deep Recovery. Day 21 of 42. Two of four steps completed today.")`.
- Primary CTA: `.accessibilityHint("Opens today's remaining protocol steps.")`.
- Considered rows: each is a button with label combining name + rationale + duration.
- Browse escape: button, label "Browse all protocols including locked ones.".
- Previous echo: `.accessibilityLabel("Previous week note: you skipped the breathwork step twice. We adjusted around it.")`.
