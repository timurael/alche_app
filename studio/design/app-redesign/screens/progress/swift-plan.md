# Progress — Swift Implementation Plan

## What changes (high level)

`WellnessProgressView` drops from a four-card dashboard to a single narrated chart. Alche picks the metric worth talking about; the member can override via a mono footer row.

Top to bottom:

1. Top bar: back + `DataSourceIndicator("Sample Data")`.
2. `AlcheReadStamp` + `AlcheReadCard` — two-line opening read.
3. **Single slim chart** (140pt) showing the metric Alche picked. Inline annotation over the inflection point ("Deep Recovery · Day 1"). Dashed vertical marker at protocol start. Current-value dot.
4. Mono `alche-data-strip` under the chart — 14D AVG · DELTA · STREAK. (Same pattern as Home.)
5. `AlcheInterstitial` — narrator connects the chart to what's unresolved ("Energy didn't follow sleep up — yet. Give it another week.")
6. **Mono picker footer row** — metric options (Sleep / Mood / Energy / Overall) + range options (7D / 30D / 90D). Inline, underlined, tappable. No iOS `.segmented` picker.
7. A single **check-in continuity row** showing "12 of 14 — last night noted 'slept deep'" with a chev to full history.
8. Primary action: `Log today's check-in`.
9. Secondary: `Open full history →`.

## View structure (target)

```swift
struct WellnessProgressView: View {
    @State private var viewModel = ProgressViewModel()

    var body: some View {
        ScrollView {
            VStack(alignment: .leading, spacing: 0) {
                AlcheReadStamp(label: "ALCHE · YOUR CURVE")
                AlcheReadCard(lines: viewModel.openingRead)

                NarratedChart(
                    data: viewModel.chartData,
                    metricLabel: viewModel.selectedMetric.label.uppercased(),
                    rangeLabel: viewModel.selectedRange.shortLabel,
                    scaleLabel: viewModel.selectedMetric.scaleDescription,
                    annotations: viewModel.chartAnnotations,
                    protocolStartMarker: viewModel.protocolStartDate
                )

                AlcheDataStrip(items: viewModel.dataStripItems)
                AlcheInterstitial(voice: viewModel.unresolvedLine)

                ProgressPickerFooter(
                    selectedMetric: $viewModel.selectedMetric,
                    selectedRange: $viewModel.selectedRange,
                    metricOptions: ProgressMetric.allCases,
                    rangeOptions: ProgressRange.allCases
                )

                CheckinContinuityRow(
                    completedCount: viewModel.checkinCompletedCount,
                    totalDays: viewModel.selectedRange.days,
                    latestNote: viewModel.latestCheckinNote
                )

                AlchePrimaryAction(label: "Log today's check-in") { viewModel.openLogSheet() }
                AlcheSecondaryAction(label: "Open full history") { viewModel.openHistory() }
            }
        }
        .background(Color.alcheBackground)
        .toolbar {
            ToolbarItem(placement: .topBarTrailing) {
                DataSourceIndicator("Sample Data")
                    .opacity(viewModel.isUsingMockData ? 1 : 0)
            }
        }
        .navigationBarTitleDisplayMode(.inline)
    }
}
```

### Sub-views

- `NarratedChart` — `Chart { LineMark + RuleMark(protocolStart) + PointMark(current) }` using Swift Charts. Single plot. Axis: 3 horizontal gridlines, min date / middle date / current date X labels. Annotation rendered as a `.chartOverlay` positioned at the annotation's date with a bordered mono tag + 14pt drop-line.
- `AlcheDataStrip(items:)` — shared component. Mono row of `k: v` pairs separated by 3px dots.
- `ProgressPickerFooter` — two `HStack`s of `UnderlineChoice` items. Each item is a `Button` with mono label; selected shows a 1.5px underline + bold. NOT a `Picker`.
- `CheckinContinuityRow` — `AlcheUtilityRow` variant. Kicker "CHECK-INS · LAST 14 DAYS", title "12 of 14 — last night noted 'slept deep'".

## ViewModel changes

Edits to `ProgressViewModel`:

- Add `openingRead: [String]` — generated by `ProgressNarrator`. Max two lines. First line names the biggest positive movement; second line names what's holding or worth watching.
- Add `chartAnnotations: [ChartAnnotation]` — events overlaid on the chart (protocol starts, GlowScan dates). Deduplicated, max 1 rendered inline.
- Add `protocolStartDate: Date?` — date of current active protocol start, within selected range.
- Add `unresolvedLine: String` — interstitial voice. Highlights a metric that did NOT move expected. From `ProgressNarrator.unresolved(from: summaries)`.
- Add `dataStripItems: [DataStripItem]` — `[(k: "14D AVG", v: "3.9"), (k: "DELTA", v: "+0.5"), (k: "STREAK", v: "12D")]`.
- Add `recommendedMetric: ProgressMetric` — Alche's pick. On first load, `selectedMetric = recommendedMetric` unless member manually overrode (persisted in UserDefaults).
- Add `latestCheckinNote: String?` — first non-empty recent note for the continuity row.
- Keep: `currentAverage`, `trend`, `streakDays`, `filteredCheckins`, `chartData`, `isUsingMockData`.

## Data dependencies

- Needs `recommendationScore` logic — weighted by `|delta|` + goal alignment. Pure function, unit-testable.

## Accessibility notes

- Chart: `.accessibilityLabel("Sleep over 30 days, on a 1 to 5 scale. Started Deep Recovery protocol on April 8. Current value 4.4, up from 3.5 at the start of the range.")`.
- Read lines: combined label reading full sentences.
- Picker footer: each `UnderlineChoice` is a `Button` with `.accessibilityAddTraits(.isSelected)` when active.
- Data strip: each k/v pair combined, e.g. "Fourteen-day average 3.9. Delta plus 0.5. Streak 12 days."
- Continuity row: "Check-ins last 14 days. 12 of 14 logged. Latest note: slept deep. Button, opens history.".
- Dynamic Type tested; annotation tag position recalculated via `.chartOverlay` reader.
