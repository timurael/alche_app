# Biomarkers — Swift Implementation Plan

## What changes (high level)

`BiomarkerDashboardView` is fully rewritten to a dialog-first layout:

1. A single top-right `DataSourceIndicator("Sample Data")` — the duplicate at the bottom is removed.
2. An **opening read** (2 italic lines) delivered by `AlcheReadCard` — frames the panel as a letter, not a dashboard.
3. A **bio-age hero strip** (italic number + mono delta chip + humanizing sub-line) instead of the current card.
4. An **`.alche-interstitial`** between bio-age and markers — Alche narrates the transition.
5. A **"Two to look at"** block showing only the markers Alche chose to highlight (max 2). Each row has a muted italic context line ("Lower than last spring. Within optimal.") — no warning triangles.
6. A **"Quietly fine —"** mono chip line replacing the CATEGORIES card list.
7. One primary action (book follow-up panel) + one secondary (connect lab).
8. A footer mono continuity strip: "Last panel · 12 weeks ago · 18 markers".

If `viewModel.isUsingMockData == true` and no real panel is connected, the entire marker section collapses to a "preview" state (desaturated, single narration: "We haven't read your blood yet").

## View structure (target)

```swift
struct BiomarkerDashboardView: View {
    @State private var viewModel = BiomarkerViewModel()

    var body: some View {
        ScrollView {
            VStack(alignment: .leading, spacing: 0) {
                topBar
                AlcheReadStamp(label: "ALCHE · YOUR PANEL TODAY")
                AlcheReadCard(lines: viewModel.narrativeLines)
                bioAgeHero
                AlcheInterstitial(voice: viewModel.transitionLine)
                BiomarkerWatchSection(markers: viewModel.topTwoMarkers,
                                      comparisonDate: viewModel.lastPanelDateLabel)
                BiomarkerHeldChips(held: viewModel.heldSteady)
                primaryCTA
                secondaryCTA
                footerContinuity
            }
        }
        .background(Color.alcheBackground)
        .toolbar {
            ToolbarItem(placement: .topBarTrailing) {
                DataSourceIndicator("Sample Data")
                    .opacity(viewModel.isUsingMockData ? 1 : 0)
            }
        }
        .navigationTitle("Biomarkers")
        .navigationBarTitleDisplayMode(.inline)
    }
}
```

### Sub-views

- `bioAgeHero` — `VStack(alignment: .leading)` with mono label, then `HStack` of italic Newsreader display (`viewModel.biologicalAgeString`) + bordered mono delta chip, then italic Newsreader sub-line (`viewModel.bioAgeHumanLine`).
- `BiomarkerWatchSection` — header with label + comparison mono tag, followed by up to 2 `BiomarkerWatchRow`s. No warning triangles. Each row: name + italic context + mono value + mono optimal range.
- `BiomarkerHeldChips` — lead text "Quietly fine —" in italic, then a `FlowLayout` of up to 4 category chips and a "+N MORE" chip. Tap opens full category browser.
- `footerContinuity` — single centered mono line, separator above.

## ViewModel changes

Edits to `BiomarkerViewModel`:

- Add `narrativeLines: [String]` — generated by `BiomarkerNarrator.opening(from:)`. Deterministic, appearance-based. Max 2 lines.
- Add `transitionLine: String` — Alche's interstitial voice. Connects bio-age insight to marker action.
- Add `topTwoMarkers: [AttentionMarker]` — curated (max 2). Algorithm: worst deviation + largest delta since last panel.
- Add `heldSteady: [CategorySummary]` — categories with all markers in range, sorted by stability.
- Add `bioAgeHumanLine: String` — humanizing one-liner ("two and a half years younger than…"). From `BiomarkerNarrator.bioAgeContext(delta:)`.
- Add `lastPanelDateLabel: String` — formatted "PANEL · FEB 2026".
- Keep: `profile`, `attentionMarkers`, `categorySummaries`, `biomarkers`, `isUsingMockData`.
- Remove branch where `isUsingMockData` renders normal numeric UI. When `true` and no real panel linked, route to the **preview state** (see below).

### Preview state (mock, no connected panel)

```swift
if viewModel.isUsingMockData && !viewModel.hasConnectedPanel {
    BiomarkerPreviewEmptyState(
        lines: ["We haven't read your blood yet.",
                "This is what your dashboard will look like,",
                "once we have."],
        primary: .connect, secondary: .learnMore
    )
}
```

The preview state wraps the existing dashboard in `.opacity(0.5)` + `.disabled(true)` with a "PREVIEW" mono overline at top.

## New / renamed components

| Component | Purpose |
|-----------|---------|
| `AlcheReadCard(lines:)` | Reusable italic read block (shared across all dialog-first screens). |
| `AlcheReadStamp(label:)` | Reusable 4-dot stamp + label row. |
| `AlcheInterstitial(voice:)` | Renders the `.alche-interstitial` pattern — hairline-top-and-bottom italic voice. |
| `BiomarkerNarrator` | Pure utility. Builds `narrativeLines`, `transitionLine`, `bioAgeHumanLine`. Vocabulary-gated (no "indicates," "demonstrates," "diagnosed"). Unit-tested. |
| `BiomarkerWatchRow` | Replaces `AttentionMarkerRow`. No warning icon. Italic context line. |
| `BiomarkerHeldChips` | Flow-layout mono chip row with lead italic text. |

Components marked "shared" should live at `app/Alche/Design/Components/Dialog/*.swift`.

## Data dependencies

- Existing: `BiomarkerProfile`, `Biomarker`, `AttentionMarker`, `CategorySummary`.
- New computed fields on `BiomarkerViewModel` (listed above).
- `BiomarkerNarrator` is pure — takes model types in, returns strings out. No services.

## Accessibility notes

- Whole read block grouped with `.accessibilityElement(children: .combine)`.
- Bio-age: combined label "Biological age 31.4, 2.6 years younger than chronological."
- Watch rows: combined label "Fasting glucose, 78 milligrams per deciliter. Lower than last spring. Within optimal. Button."
- Chips: each is a button with label "Iron, held steady. Tap for details."
- Dynamic Type tested up to `.accessibility3` — italic Newsreader lines use `.font(.alcheDisplayM)` which scales via existing token mapping.
- Color: delta chip blue (`Color.alcheAccentPrimary`) meets 4.5:1 on white.

## Compliance gates

- [x] **One** `DataSourceIndicator` — top-right trailing toolbar only. Remove both current indicator instances from the old view body.
- [x] `BiomarkerNarrator` vocabulary whitelist enforces non-clinical language: no "indicates," "demonstrates," "levels," "diagnosed," "treats." Swift test: `BiomarkerNarratorVocabularyTests` with a failing example for each banned word.
- [x] Every numeric render is prefaced or accompanied by narrative context (per-row italic context line).
- [x] Sharp 2px radii only. Hard drop shadow for primary CTA.
- [x] No emoji. Unicode `→` for arrows.
- [x] Preview state present for mock-no-panel path.
