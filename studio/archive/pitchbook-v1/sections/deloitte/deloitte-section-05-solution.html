alche | Pre-Seed Investment Memorandum
The Solution
Four integrated layers, each creating demand for the next -- and each generating revenue.
The KDGB framework (Know-Do-Get-Belong) is a business process, not a brand concept. Each layer has a user action, a revenue lever, and a retention mechanism.
01
KNOW
User tracks biomarkers, wearable data, biological age. Longitudinal data creates personalization engine.
Revenue: Subscription (EUR 19-99/mo)
02
DO
User follows personalized protocols -- nutrition, recovery, movement. AI-guided, physician-informed.
Revenue: Service bookings, protocol access
03
GET
User buys protocol-linked supplements, functional products. Recommended by data, not generic catalog.
Revenue: Product margins (40-65%)
04
BELONG
User joins accountability circles, shared challenges, events. Community as longevity intervention.
Revenue: Events, premium community access
KNOW → DO → GET → BELONG → KNOW (repeat, each cycle stickier)
Layer User Action Revenue Lever Retention Mechanism Key Metric
KNOW Biomarker dashboard, wearable sync Subscription (EUR 19-99/mo) Longitudinal data accumulates; 6 months > snapshot Activation rate

Each step creates demand for the next. KNOW creates motivation to DO. DO creates need to GET. GET creates belonging through shared rituals. BELONG generates better data.
Why integration is the structural advantage
Single-vertical competitors own one layer. Oura owns wearable data. AG1 owns a supplement. Restore owns recovery. Nobody connects all four. Integration is structurally hard to bolt on because it requires different organizational DNA than single-vertical excellence. A competitor would need to rebuild from the ground up -- the same thing alche is doing, but without the greenfield advantage.
DATA TRUST: Only 8% of EU consumers share health data (BEUC 2024). GDPR-native architecture, transparent data policies, user-controlled sharing. In a market where trust is the scarcest resource, privacy-first design is a competitive advantage.
