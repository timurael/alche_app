The Data Feedback Loop

The entire system is built around a feedback loop that connects physical and digital layers. This is the core contribution: a continuous data-driven feedback loop.

1. AI Glow Scan + Goal Selection
→
2. Personalised Protocol Created
→
2A. Doctor validates and enhances protocol (CGM users)
→
3. Booking (LED + smoothie pre-order)
→
4. Experience (physical visit)
5. Data Captured (mood, energy, skin, purchases)
→
5A. CGM Glucose Stream (continuous biometric data)
→
6. Insights Generated (patterns + glucose correlations)
→
7. Recommendations (replenishment, events, protocol adjust)
→
7A. Doctor Review (medical-grade adjustment based on CGM + all data)
→
8. User Re-Entry (smarter protocol, continuous learning)
The doctor does not slow down the loop. Steps 2A and 7A operate asynchronously. The user continues through the loop without interruption while the doctor reviews data and refines the protocol in the background. When an update is ready, the user receives a notification: "Dr. Jarmoukli has updated your protocol. View your new wellness plan." This architecture transforms alche from a simple wellness app into a medical-grade wellness intelligence platform.
